# 配置文件说明

## 配置文件概述

FDX SMART WORK 2.0 项目包含多个配置文件，用于管理不同环境的设置、构建选项、代码质量工具等。

## 核心配置文件

### 7. 环境变量配置

#### .env.example
```bash
# 应用配置
NEXT_PUBLIC_APP_NAME=FDX SMART WORK 2.0
NEXT_PUBLIC_APP_VERSION=2.0.0
NEXT_PUBLIC_APP_URL=http://localhost:3000

# 数据库配置
DATABASE_URL=postgresql://username:password@localhost:5432/fdx_smart_work

NEXT_PUBLIC_SUPABASE_URL=your-supabase-url
```

## 配置文件管理最佳实践

### 1. 环境变量管理
- 使用 `.env.example` 作为模板
- 敏感信息不提交到版本控制
- 不同环境使用不同的配置文件

### 2. 配置验证
```typescript
// lib/config.ts
import { z } from 'zod'

const configSchema = z.object({
  NODE_ENV: z.enum(['development', 'production', 'test']),
  DATABASE_URL: z.string().url(),
  JWT_SECRET: z.string().min(32),
  NEXT_PUBLIC_APP_URL: z.string().url(),
})

export const config = configSchema.parse(process.env)
```

### 3. 配置热重载
```javascript
// next.config.js
const nextConfig = {
  webpack: (config, { dev }) => {
    if (dev) {
      config.watchOptions = {
        poll: 1000,
        aggregateTimeout: 300,
      }
    }
    return config
  },
}
```
